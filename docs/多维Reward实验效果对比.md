# 多维 Reward v1 引导 MCTS 实验效果对比

本文只记录当前 v1 版本的实验效果：阶段化五维 reward、五维 Q 回传、阶段化 target dimension 和 `final_reward_weights` 排序。文中不讨论 v1 之外的额外改动。

## 一、实验设置

| 配置项 | Baseline | 多维 Reward v1 |
|--------|----------|------------------|
| 节点选择 | 标量 UCT：$Q + C \cdot U$ | 多维加权 UCT：$Q_{multi} + C(t) \cdot U$ |
| 探索系数 $C$ | 固定 0.70 | Cosine 自适应衰减：1.0 → 0.5 |
| Reward 形式 | 单一 `mcts_reward` | effectiveness / diversity / stability / turnover / overfit 五维 reward |
| 搜索阶段 | 不区分阶段 | early / middle / late 阶段化权重 |
| 最终排序 | `mcts_reward` | `final_reward_weights` 加权排序 |
| 搜索预算 | 24 步/轮 | 24 步/轮 |
| 总轮数 | 20 | 20 |
| 训练期 | 2010-01-01 ~ 2019-12-31 | 相同 |
| 验证期 | 2020-01-01 ~ 2025-04-30 | 相同 |

多维 Reward v1 的含义：MCTS 不再只用单一标量 reward 评价节点，而是同时维护五个维度的质量信号。前期偏向 diversity / overfit，鼓励打开结构空间；中期五维相对均衡；后期偏向 effectiveness / stability，配合较低的 cosine 探索系数集中开发更可能入库的表达式。

## 三、改进分析

### 3.1 入库数量和训练期效率明显提升

Baseline 在 20 轮中入库 56 个因子，多维 Reward v1 入库 81 个因子，产出提升约 44.6%。更关键的是训练期通过率从 38.4% 提升到 56.7%，说明多维 reward 对 MCTS 搜索方向的引导是有效的：它不是简单增加候选数量，而是提高了搜索过程触达有效候选的概率。

从日志看，多维 Reward v1 在成功轮次中的训练期表现更稳定：

- Cycle 12：20 → 14 个因子通过训练期筛选
- Cycle 15：18 → 16 个因子通过训练期筛选
- Cycle 19：20 → 18 个因子通过训练期筛选
- Cycle 20：20 → 12 个因子通过训练期筛选

这说明阶段化权重在后期偏向 effectiveness / stability 后，确实能把搜索预算更多集中到更容易通过指标筛选的表达式区域。

### 3.2 平均预测能力和收益表现提升

多维 Reward v1 的平均 |IC| 从 0.0545 提升到 0.0600，平均 |IC20| 从 0.0642 提升到 0.0679，说明新策略挖到的因子整体方向性更强。

收益侧也有提升：平均超额收益从 4.31% 提升到 5.90%，正超额收益占比从 78.6% 提升到 87.7%。平均 Long Sharpe 和 LS Sharpe 也分别提升到 0.539 和 2.706。

这说明多维 Reward 的收益并不只体现在 `mcts_reward` 这类内部评价指标上，也能传导到样本外入库后的实际收益指标。

### 3.3 极值因子没有超过 Baseline

多维 Reward v1 的平均表现更好，但最优单因子没有超过 Baseline：

- Baseline 最佳超额收益：22.0%
- 多维 Reward v1 最佳超额收益：17.5%
- Baseline 最佳 Long Sharpe：1.633
- 多维 Reward v1 最佳 Long Sharpe：1.396

这说明 v1 更偏向提高批量产出和平均质量，而不是发现更强的极端 top 因子。多维 reward 的加权机制会抑制过度偏科的节点，因此可能减少了极端高收益但结构或稳定性不够均衡的候选。

### 3.4 结构搜索方向发生明显变化

多维 Reward v1 明显改变了表达式结构分布。Baseline 中主要结构是 Corr / Mean / WMA / Rank 组合，而多维 Reward v1 几乎全部集中到 Cov / Std / WMA / Log / Abs 结构：

| 结构项 | Baseline | 多维 Reward v1 |
|--------|:---:|:---:|
| `Corr` | 23/56 | 1/81 |
| `Cov` | 0/56 | 81/81 |
| `Mean` | 47/56 | 0/81 |
| `Std` | 18/56 | 81/81 |
| `amount` 字段 | 30/56 | 0/81 |
| `close` 字段 | 56/56 | 81/81 |
| `volume` 字段 | 55/56 | 81/81 |

这说明多维 Reward v1 确实让搜索跳出了原先以 Corr / Mean 为主的结构区域。但 v1 仍未解决结构集中问题：新因子高度偏向 `Cov(Delta(close), Delta(volume))` 结构，字段和算子层面的多样性不足。

### 3.5 稳定性与换手存在代价

多维 Reward v1 的平均 `effectiveness_score` 从 0.576 提升到 0.686，说明有效性维度提升明显。但 `stability_score` 从 0.941 下降到 0.893，`turnover_proxy` 从 0.101 上升到 0.169，说明 v1 在追求有效性和产出数量时，引入了更高换手和更弱稳定性。

这与当前 `final_reward_weights` 的设置一致：最终排序中 effectiveness 权重最高，turnover 权重较低，overfit 未显式进入最终排序权重。因此，v1 的主要收益来自 effectiveness 导向增强，代价是换手和稳定性控制不足。

## 五、结论

多维 Reward v1 的核心价值是**提升搜索的有效候选命中率和批量入库能力**。在相同 20 轮实验中，入库因子数从 56 提升到 81，训练期通过率从 38.4% 提升到 56.7%，平均 |IC|、平均超额收益、平均 Long Sharpe 和平均 LS Sharpe 均有提升。

但当前版本仍存在两个明显不足：

1. **极值能力不足**：最佳超额收益和最佳 Long Sharpe 没有超过 Baseline，说明 v1 更擅长提高平均质量，而不是寻找最强 top 因子。
2. **结构集中问题明显**：新因子高度集中在 `Cov(Delta(close), Delta(volume))` 结构，说明 v1 只把多维反馈接入搜索决策，并没有额外处理字段或算子层面的多样性。

整体看，多维 Reward v1 是一次有效的最小化改进。它验证了一个核心结论：将已有五维反馈接入 MCTS 的节点选择、target dimension 抽样、回传和最终排序，可以显著提高有效候选命中率和平均质量；但 v1 本身不解决 top 因子上限、结构集中和换手上升问题。
